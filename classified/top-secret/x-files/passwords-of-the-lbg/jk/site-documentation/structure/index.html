How we structure our site.

Because we are hosted using GitHub Pages, we can only serve static content on our website. This effects how we structure the pages on our site and hence how we structure the files and folders in the codebase.

Static sites

As we are hosted using GitHub pages, we can only serve static content on our site. This means there is a direct correspondence between how files and folders are structured and the links to the pages on the website.

If we create a file index.html in a folder hello, we can find the corresponding page using the link bestghent.eu/hello/. We could also create two pages in the same folder, like hello1.html and hello2.html, but then we get the less aesthetically pleasing bestghent.eu/hello/hello1.html and bestghent.eu/hello/hello2.html. This makes index.html special, as it is the default page to be loaded when no file is specified. On this website, we only use index.html files to keep our links nice and tidy. This means we create seperate folders for every page we create.
