Git & VS Code setup

To make changes to the website, you'll need to get a code editor (unless you feel more comfortable using notepad of course). Visual Studio Code is a great code editor that provides a lot of features and has a vast library of free extensions.
Get started with VS Code

Now that you can edit the site, you still need to get your hands on its code. Cloning/pulling (downloading) and pushing (uploading) changes to GitHub can be done using the version control system called Git. VS Code provides a great interface to Git, but you are free to use any Git client you like. Personally, I like to use fork.

Code

This site, like most websites you see online, is based on 3 languages: HTML, CSS and JavaScript.
Html describes the layout of the site, like an advanced word document.
CSS will apply styling to all elements in a page, like coloring buttons, or setting the font of the text.
JavaScript does everything else. It is the only real programming language in this trinity and is responsible for everything the previous two can't do, like dynamically changing content, or reacting to user input (e.g. for games).
How to make a new page Code-style guidelines

Templates

When writing a html page from scratch, there is often a lot of boilerplate code that is repeated across almost every page (e.g. setting the page icon to the BEST logo). To avoid having all of this code repetition, a template system was created using JavaScript. It condenses all of the boilerplate down into a minimal amount of code required to boot up the system.
How the template-system works

Another advantage of templates is that they allow for the automatic generation of new elements. This is for example how the navigation works: instead of copying the entire navbar into every page, it was made into a template that could be included in one line.

Along with the template system, we also made a few CSS templates to quickly make buttons, slides, ... The button below links to a summary of all of these templates. They make creating a new page a lot easier, so maybe read through them at least once.
