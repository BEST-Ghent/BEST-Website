How to make a new page.

This page contains everything you need to know to make a new page from scratch.

Make a new index.html file

Every page is described by a .html file describing its markup (HTML=HyperText Markup Language). To learn where to create this file, check out the documentation about file structure.

There are two main types of pages we can create on this website: regular pages and Fullpage pages. Regular pages are, like the name suggests, pretty standard. Fullpage pages make use of the Fullpage plugin. They are used on the homepage of our website and the documentation.

Fullpage pages require slightly different boilerplate than regular ones, so this page is split in two. The left halves are for regular pages and the ones on the right are for Fullpage pages.

Header boilerplate

All HTML files start with boilerplate code describing some general stuff about the site, like the title, the icon, what scripts and stylesheets to import,...

On the first line, we specify that the document is supposed to be an HTML file. This is mainly to let browsers know what kind of document they're dealing with.

Next, we open the <html> tag. This tag is the root of our document and will contain every other element in our code. Note that we also specify class="fouc-barrier". This refers to the fix for a so-called FOUC:

When a browser recieves a page, it will (almost) immediately start to render that page. If the contents of that page are later changed (f.e. using JavaScript), it will update its rendering to reflect that change. The problem is that the template system only activates when the page is (almost) ready to be rendered, meaning that their will be a short moment where the page is technically unfinished but already showed to the client. This causes a quick flash when the template system eventually catches up and displays the correct content.

To prevent this, the entire page is set to not display anything by default. This is done using a so-called FOUC-barrier with it's styling set to display: none;. Because this barrier needs to block the entire page, this class is added to the <html> tag. When the template system is finished, it removes the FOUC-barrier, revealing the page.

After this, we start the <head> tag. The head of a page contains metadata describing information about the page, like the title and icon the browser displays. While the title of a page can differ from page to page, the icon usually doesn't. This is why metadata like the icon is inserted by the template-system. This means we have to add the class="templates-head" class to the system tag. The title is still specified manually as it can theoretically differ from page to page (though we use BEST Ghent everywhere).

The only two other things that need to go into the head are the inclusion of the template system itself and the styling for the FOUC-barrier. Everything else that usually needs to be in a head is then included by the template system. This includes other scripts and Stylesheets.

After we close the head we open the <body> tag. The body of a page contains the page itself. At this point, a general page and a Fullpage page start to differ.

General

For a general page, we are technically done. Their is just one thing we should add before starting the rest of the page and that's a style. By adding a style class to the body, we define the color theme of the page. For a full list of available styles, check out the list of features. If you don't specify a style, some elements like the copyright section at the bottom of a page won't render correctly.

Fullpage

A Fullpage page does not need a style specified as most slides usually have a different one. That's why the body has no extra classes. What we do need to add is a <div> that will contain every Fullpage slide. This div gets id="fullpage" such that the Fullpage script can find it and link to it correctly. 

<!DOCTYPE html>

<html class="fouc-barrier">

<head class="templates-head">
	<title>BEST Ghent</title>
	<script src="/static/scripts/templates.js" type="module" async="false"></script>

	<!-- This page loads stylesheets with javascript, causing FOUC's. To prevent this, we add this FOUC barrier (which should also be added as a class in the <html> tag).
		When the templates system is finished, it will automatically remove the barrier, making the browser render the page. -->
	<style type="text/css">.fouc-barrier { display: none; }</style>
</head>

<body class="style-green">
<!DOCTYPE html>

<html class="fouc-barrier">
    
<head class="templates-head">
    <title>BEST Ghent</title>
    <script src="/static/scripts/templates.js" type="module" async="false"></script>

    <!-- This page loads stylesheets with javascript, causing FOUC's. To prevent this, we add this FOUC barrier (which should also be added as a class in the <html> tag).
		When the templates system is finished, it will automatically remove the barrier, making the browser render the page. -->
	<style type="text/css">.fouc-barrier { display: none; }</style>
</head>

<body>
    <div id="fullpage">

Footer boilerplate

Sadly, every page must come to an end eventually. To give the browser and the client the closure they deserve, we need to properly end the page. This means closing the </html> and </body> tag and adding scripts and navigation.

The class="templates-scripts" template will include all the JavaScript we need to run the page. This includes the Fullpage script, but also the owl-carousel plugin, the scripts that display our calendars on event pages, ...

For the navigation specific stuff below: To specify what the contents of this navigation should be, you set the data-nav-desc attribute to a navigation description. All navigation descriptions are defined in the templates.js file. To learn more about them, check out this page.

General

To add navigation to a general page, you can use the class="templates-nav-main" template. This template will create a navigation bar at the top of the page.

Finally, we want to add a small copyright notice at the bottom of every page. This can be done by creating a <section> tag with the class="templates-copyright" template. The copyright notice serves as a reminder that people can't just use foto's or other content from our website without asking for permission.

Fullpage

For Fullpage pages, you can use the class="templates-nav-main" template. This navigation will follow sections when scrolled, making it always visible.

We don't have a copyright notice on our Fullpage pages at the moment, as they usually don't contain foto's that are that valuable. If you ever need to, you can of course add one yourself.

We do still need to close the Fullpage div though, hence the </div>. 

    <section class="templates-copyright"></section>

    <div class="templates-nav-main" data-nav-desc="documentation-main"></div>

	<!-- Placeholder for javascript imports (imported using the templates.js file) -->
	<div class="templates-scripts"></div>
</body>

</html>
    </div>

    <div class="templates-nav-fullpage" data-nav-desc="documentation-home"></div>

	<!-- Placeholder for javascript imports (imported using the templates.js file) -->
	<div class="templates-scripts"></div>
</body>

</html>

How to add sections

As you may have noticed, all content on this website is divided into sections. Whilst the creation of a section differs between general and Fullpage sections, the way their contents works is usually the same:

First we wrap everything in a <div class="container">. This will provide the nescessary padding and margins to the sides of its content to make sure your content always has a page-like appearance, no matter the size of your display. You can of course check out what it would look like without this container by removing the class. The effect is most notable on wider displays.

Next we start a row using <div class="row">. Every row can be divided up into columns. The width of a column is defined in terms of 1/12ths of the entire row width.

This means that if you want 1 column to fill the entire row, you use <div class="col-sm-12">.
If you want 2 columns to divide the entire row equally, you use 2 times <div class="col-sm-6">.
If you want a column with text and a smaller one with an image, you can use something like <div class="col-sm-9"> for the text and <div class="col-sm-3"> for the image.
etc.

To get more examples of how to use rows vs columns, you can check out pages that use it extensively, like this one. Once your section has been set up, you can write anything you want in it. If you are not familiar with HTML, you can copy stuff you like from other pages.

Probably the most interesting tags to use are: <h1>, <b>, <br> and <p>.
<h1> declares that its content is a title. The 1 ranks its importance. <h2> will appear smaller than <h1>, ...
<b> marks its content as bold. This boldness can also change color when used in <h1> based on the style of the page.
<br> inserts a line break. This is one of the few tags in HTML that does not need to be closed with </br>.
<p> defines a paragraph. Since HTML ignores linebreaks by default, you have to use <p> to split up your text into paragraphs.

General

For most sections, it is sufficient to just start a <section> tag.

If you want a section to stand out among the others, you can add the class="section-odd" class. This will make the section slightly gray, like this one.

Fullpage

A Fullpage section requires some more work. First the plugin needs to know that the section is supposed to become a Fullpage section. This is done by adding the class="section" to the <section> tag.

Since Fullpage pages have no overarching style, we also need to specify the style of the section (in this case sec-blue)

Finally, if we want to be able to navigate through our sections, we need to specify the data-anchor attribute. The value of this attribute has to correspond to the link in the navigation description. For example: The navigation description for the homepage of this documentation contains an entry "Tools":"#tools". This corresponds to a slide with data-anchor="tools" (#tools without the #) and a navigation widget that says Tools. 


<section>
    <div class="container">
        <div class="row">
            <div class="col-sm-12">
                <h1>Amazing section <b>title</b></h1>
            </div>
        </div>
    </div>
</section>

<section class="section sec-blue" data-anchor="section1">
    <div class="container">
        <div class="row">
            <div class="col-sm-12">
                <h1>Amazing section <b>title</b></h1>
            </div>
        </div>
    </div>
</section>

Hero sections

A hero section is the first thing you see on a page. This usually also makes it the first section on a page. To create a hero section, you can add class="hero" to the section tag. This will take care of some formatting, like centering the text, increasing the <h1> font size, ...

The following code is the code from the hero section of this page. 


<section class="hero sec-img" style="background-image: url('static/media/code/make-a-page/make-a-page_vibe_img.png');">
    <div class="overlay-black">
        <div class="container">
            <div class="row">
                <div class="col-sm-12">

                    <h1>How to make a <b>new</b> page.</h1>
                    <p>
                        This page contains everything you need to know to make a new page from scratch.
                    </p>
                </div>
            </div>
        </div>
    </div>
</section>

Note how the code above also shows how the background image is set. The class="overlay-black" is also a part of this. To learn more about these features, check out the list of features page.

Everything else

To learn how to use all the widgets, gadgets, features, ... that are premade for you, check out this page. You can of course also make something entirely new. If you have the coding skills to do so: great! If not: There are plenty of resources online to acquire them:

  1. W3Schools: Contains a lot of tutorials about HTML, JavaScript, CSS and more.
  2. General googling: This is still a fast way to discover things programming related*.
  3. Stack Overflow: Has all the questions about everything programming related. It also has all possible answers (the good, the bad and the ugly).
  4. AI: Can be a powerfull tool when used correctly. Please think about the environmental and cultural impact with every prompt and try to use it for inspiration on how to do something rather than copying its answers directly.

*Googles AI overview draws a lot of energy and gives too many bogus results to be reliable (at least for programming IMHO). If you want, you can disable most of it by including -ai somewhere in your prompt.
